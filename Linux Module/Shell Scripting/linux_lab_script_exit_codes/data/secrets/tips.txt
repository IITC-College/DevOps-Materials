# Script Exit Codes Lab - Optional Tips

## Level 1 Clue 1 (exit 0/1)
- if [ -d "$TARGET" ]; then echo "OK"; exit 0; else echo "FAIL"; exit 1; fi
- TARGET="${1:-../data/environments/project1}" – default if no argument.
- echo $? right after running the script shows the exit code (0 or 1).

## Level 1 Clue 2 (check config)
- CONFIG="${1:-../data/configs/app.conf}"; if [ -f "$CONFIG" ]; then echo "Config found."; exit 0; else echo "Config not found."; exit 1; fi

## Level 1 Clue 3 (&& and ||)
- ./script.sh && echo "Success" – second command runs only if first exited 0.
- ./script.sh || echo "Failed" – second command runs only if first exited non-zero.

## Level 2 Clue 1 (chain steps)
- Run step 1; CODE=$?; if [ $CODE -ne 0 ]; then echo "Step 1 failed."; exit 1; fi; run step 2; CODE=$?; if [ $CODE -ne 0 ]; then exit 1; fi; exit 0.

## Level 2 Clue 2 (wrapper pass-through)
- ./inner_script.sh; CODE=$?; echo "Exit code: $CODE"; exit $CODE – caller sees same exit code.

## Level 2 Clue 3 (pipeline)
- Validate first; if failed, exit 1 (do not run deploy). Deploy only if validate passed. CI/CD uses exit codes for green/red and to abort on failure.
