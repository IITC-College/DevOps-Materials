          "Cross-Department Leak" - Fixing Marketing File Ownership
================================================================================

Welcome to Exercise 2!

The marketing department can't access their own files because they're owned
by engineering users. Meanwhile, engineering users can read marketing's
campaign plans and analytics data!

BACKGROUND:
-----------
Sometimes a quick fix needs TWO changes. Consider the chgrp command as an
alternative to chown for changing just the group:

  sudo chgrp marketing file.txt     → Change group to marketing
  sudo chown manager file.txt       → Change owner to manager
  sudo chown manager:marketing file → Change both at once

EXERCISE:
---------
Fix the ownership of BOTH marketing files.

INSTRUCTIONS:
-------------
1. Check the current state of marketing files:

   ls -l company/marketing/

2. Notice the problems:
   - campaign_plan.txt is owned by developer:engineering
   - analytics_report.csv is owned by intern:engineering

   Marketing team members can't access their own files!

3. Check who is in the marketing group:

   getent group marketing

   The manager is in the marketing group.

4. Fix campaign_plan.txt - change owner to manager AND group to marketing:

   sudo chown manager:marketing company/marketing/campaign_plan.txt

5. Fix analytics_report.csv using TWO separate commands this time:
   First change the owner:

   sudo chown manager company/marketing/analytics_report.csv

   Then change the group:

   sudo chgrp marketing company/marketing/analytics_report.csv

6. Verify both fixes:

   ls -l company/marketing/

   Both files should now show "manager marketing".

7. Check the permissions make sense:
   - campaign_plan.txt: 664 (rw-rw-r--) → manager writes, marketing reads, others read
   - analytics_report.csv: 644 (rw-r--r--) → manager writes, everyone reads

COMMANDS TO USE:
----------------
- ls -l company/marketing/
- getent group marketing
- sudo chown manager:marketing company/marketing/campaign_plan.txt
- sudo chown manager company/marketing/analytics_report.csv
- sudo chgrp marketing company/marketing/analytics_report.csv
- ls -l company/marketing/

CHOWN vs CHGRP - WHEN TO USE WHICH:
-------------------------------------
Use chown when:
  - Changing the owner: sudo chown manager file
  - Changing both owner AND group: sudo chown manager:marketing file

Use chgrp when:
  - Changing ONLY the group: sudo chgrp marketing file
  - You want to be explicit that only the group is changing

Both require sudo. The result is the same - use whichever is clearer
for the situation.
