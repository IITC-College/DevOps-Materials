      "The Locked Handbook" - Making Shared Documents Accessible

Welcome to Exercise 3!

The company handbook and meeting notes should be accessible to ALL employees,
but they're currently locked down. The handbook is owned by "intern:intern"
with permissions 600 (only the intern can read it), and the meeting notes
are owned by "root:root" with permissions 600.

BACKGROUND:
-----------
Shared documents need a balance:
  - They must be READABLE by all relevant employees
  - They should be WRITABLE only by authorized editors
  - The ownership and permissions must BOTH be correct

For company-wide documents, a common pattern is:
  Owner: manager (can edit)
  Group: engineering (or a company-wide group)
  Permissions: 644 (owner writes, everyone reads)

EXERCISE:
---------
Fix the shared documents so all employees can read them, but only the
manager can edit them.

INSTRUCTIONS:
-------------
1. Check the current state:

   ls -l company/shared_docs/

2. Notice the problems:
   - company_handbook.txt: owned by intern:intern, permissions 600
     → Only the intern can read the company handbook!
   - meeting_notes.txt: owned by root:root, permissions 600
     → Only root can read the meeting notes!

3. Fix the company handbook:
   a. Change ownership to manager:engineering

      sudo chown manager:engineering company/shared_docs/company_handbook.txt

   b. Change permissions to 644 (owner writes, everyone reads)

      sudo chmod 644 company/shared_docs/company_handbook.txt

4. Fix the meeting notes:
   a. Change ownership:

      sudo chown manager:engineering company/shared_docs/meeting_notes.txt

   b. Change permissions:

      sudo chmod 644 company/shared_docs/meeting_notes.txt

5. Verify BOTH changes:

   ls -l company/shared_docs/

   Both files should show: manager engineering with -rw-r--r--

6. Test that you can now read them:

   cat company/shared_docs/company_handbook.txt
   cat company/shared_docs/meeting_notes.txt

COMMANDS TO USE:
----------------
- ls -l company/shared_docs/
- sudo chown manager:engineering company/shared_docs/company_handbook.txt
- sudo chmod 644 company/shared_docs/company_handbook.txt
- sudo chown manager:engineering company/shared_docs/meeting_notes.txt
- sudo chmod 644 company/shared_docs/meeting_notes.txt
- ls -l company/shared_docs/
- cat company/shared_docs/company_handbook.txt

KEY CONCEPT - COMBINING CHOWN AND CHMOD:
-----------------------------------------
In real-world scenarios, you almost always need to fix BOTH:

  Step 1: Fix WHO owns the file    → sudo chown user:group file
  Step 2: Fix WHAT they can do     → sudo chmod NNN file

Think of it as: chown sets the "who", chmod sets the "what".

Common patterns:
  Private file:   chown user:user   + chmod 600
  Team file:      chown user:team   + chmod 660
  Shared file:    chown user:team   + chmod 644
  Public script:  chown user:team   + chmod 755
